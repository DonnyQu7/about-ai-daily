# AI 应用落地知识日报方案

## 目标

每天自动从 GitHub 获取软件测试、QA 自动化、E2E/API 测试、CI 执行流、代码质量和工程自动化相关的热点项目，经过去重、评分、分类和摘要，生成中文 Markdown、HTML 报告、SVG 海报和 JSON 归档。X/Twitter 因 API 权限和成本不稳定，第一版保留接口但不进入默认日报。

## 第一阶段 MVP

第一阶段只做稳定、低成本、可每天运行的 GitHub 链路：

1. 采集来源
   - GitHub Search API：发现最近 24 小时和最近 7 天内活跃的测试/自动化/工程质量项目。
   - RSS/Atom、Hacker News、X/Twitter：保留代码接口，但默认关闭，避免偏离软件测试行业。

2. 处理流程
   - 拉取最近 24 小时 GitHub 热点。
   - 拉取最近 7 天 GitHub 热点。
   - 基于 URL 和标题去重。
   - 用测试行业关键词、软件交付语境、GitHub 热度做规则评分。
   - 对商业营销、内容创作、游戏自动化、纯安全 Recon 等偏离目标的项目降权。
   - 分类到固定业务主题。
   - 输出 Markdown、HTML、SVG 海报和 JSON 原始结果。

3. 输出结果
   - `reports/YYYY-MM-DD.md`：Markdown 归档。
   - `reports/YYYY-MM-DD.html`：可点击跳转 GitHub 的 HTML 报告。
   - `reports/YYYY-MM-DD-poster.svg`：邮件附件海报。
   - `data/items/YYYY-MM-DD.json`：机器可读归档。

## 分类体系

日报默认使用以下分类：

- QA 自动化 / 测试 Agent
- E2E / 浏览器自动化测试
- API / 接口测试与 Mock
- 自动化执行流 / CI 编排
- 测试数据 / 环境 / DevOps
- 代码质量 / Review / 静态分析
- 测试用例生成 / LLM 辅助测试
- 工程可观测性 / 质量信号
- 开发者效率自动化

## 评分逻辑

第一版不依赖 LLM，采用可解释规则：

- 内容相关性：标题、摘要、repo 描述命中 QA、testing、Playwright、Selenium、E2E、API testing、CI、pipeline、code review 等关键词。
- 软件交付语境：命中 GitHub、GitLab、PR、repo、code、SDK、CLI、CI/CD、pipeline、test automation 等信号加分。
- 偏离目标降权：sales、marketing、CRM、content、game、Genshin、red team、recon、bug bounty 等非目标语境降权。
- 热度信号：GitHub stars、forks、open issues。
- 新鲜度：24 小时和 7 天两个窗口分别排序。

评分结果用于排序，不会完全替代人工判断。后续可以增加 OpenAI-compatible LLM 评分器，把规则评分作为候选初筛。

## 推荐后续迭代

1. 接入 LLM 摘要和“为什么对测试团队有用”的判断。
2. 增加 GitHub Star 增量跟踪，需要本地保存项目历史快照。
3. 增加飞书、Telegram、企业微信推送。
4. 增加 Web Dashboard：按类别、语言、Star、时间窗口筛选。
5. 接入 X/Twitter：只采集软件测试和 QA 自动化账号或关键词。
